# Треки (CPE / FCE / ЕГЭ), деплой и разделение контента и движков

Краткая «конституция» репозитория для себя и для правок с ИИ.

## Что есть что

| Трек | Точка входа для ученика | Сборка |
|------|-------------------------|--------|
| **CPE Prep** | Корневой **`index.html`** (Levels hub, Prep) | `npm run build` / `build:cpe` → **`publish-cpe`** |
| **Mastering B2 (FCE)** | **`fce.html`**, юниты через **`unit*.html?course=fce`** | `npm run build:fce` → **`publish-fce`** |
| **ЕГЭ** | **`ege.html`** | `npm run build:ege` → **`publish-ege`** |

**Истина для пользователя сайта** — не «весь диск репозитория», а содержимое папки **`publish-*`** после скрипта `scripts/build-netlify-track.mjs`. На FCE-сайте нет полного CPE `index.html`; на CPE могут отсутствовать части чужого трека — это норма.

## Развод контента по URL (Unit 10 и дальше)

Где нужно два продукта на одном типе материала — **две папки-хаба**, без двух наборов плиток на одном URL:

- Use of English Unit 10: **`use-of-english/unit10/fce/`** (книга) и **`use-of-english/unit10/cpe/`** (Prep). Корневой **`use-of-english/unit10/index.html`** только редиректит по `?course=`.
- Словарь Unit 10: **`unit10-vocabulary/fce/`** и **`unit10-vocabulary/cpe/`**. Корневой **`unit10-vocabulary/index.html`** — редирект.

Навигация из **`use-of-english/part1-mc-cloze/`** и **`part2-open-cloze/`** на хаб Unit 10 должна идти через **`../unit10/...`** (один уровень вверх — в `use-of-english`), а не через **`../../unit10/...`** (иначе браузер лезет в несуществующий **`/unit10/`** в корне сайта).

## Контент и движки отдельно

Ты уже делаешь это там, где возможно:

1. **Движок** — переиспользуемый JS (`part1-mc-cloze-core.js`, `part2-open-cloze-core.js`, тренажёры Reading и т.д.).
2. **Контент** — JSON / небольшие data-модули (`published*.json`, `context=…`, отдельные `*-data.js`), без копирования длинных текстов заданий в монолитный хаб.

**Правило на будущее:** новое задание того же типа — по возможности **новый JSON (или data-файл) + та же страница-оболочка**, а не новый самодостаточный лабиринт HTML. Старые страницы трогаем только когда работаешь с этой темой; массовый перенос не обязателен.

## Перед коммитом / деплоем

```bash
npm run check:tracks
```

Скрипт ловит типичные ошибки путей между треками и «лестницами» `../..` из движков Use of English. Зелёный выход — хороший минимум перед `build:cpe` или `build:fce`.

## Локальная проверка

Открытие **`file://`** полезно для быстрого просмотра, но перед выкладкой лучше проверить **собранный сайт** через локальный сервер или превью Netlify — так ближе к продакшену.
