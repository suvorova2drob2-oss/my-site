Three Netlify sites = three domains (recommended setup)
=======================================================

Goal
----
- Learners get **one URL** (only the site for their exam: CPE, EGE, or FCE).
- **Name once per domain** — stored in the browser for that site only (localStorage + cookie backup). Not in Supabase.
- **English** UI for the name gate on CPE (`index.html`). EGE hub header chrome is English; unit copy may stay bilingual.

Build outputs (from repo root)
------------------------------
  publish-cpe   →  "/" serves the full CPE app (index.html). Top-level ege.html and fce.html are **not** published.
  publish-ege   →  "/" serves ege.html; fce.html is omitted; root index.html is a **tiny redirect** to ege.html (no full CPE hub on this origin).
  publish-fce   →  "/" serves fce.html; ege.html is omitted; root index.html is a **tiny redirect** to fce.html.

Each site only contains its own exam hub HTML at the repo root; students cannot open the other hubs on the wrong domain even by guessing the filename.

Create **three** Netlify sites from the **same** Git repo, with different build settings.

  **All sites:** the static tree is produced by **Vite** (`npm run vite:build` → `dist/`), then split into `publish-*`.

  Site A (CPE):  Build:  npm ci && npm run build
                 Publish: publish-cpe
                 (same as repo `netlify.toml`)

  Site B (EGE):  Build:  npm ci && npm run build:ege
                 Publish: publish-ege

  Site C (FCE):  Build:  npm ci && npm run build:fce
                 Publish: publish-fce

This repo’s netlify.toml defaults to CPE only; override build + publish in the Netlify UI for sites B and C.

Local preview: `npm install` then `npm run dev` (Vite dev server). Full publish tree: `npm run build:cpe` then serve `publish-cpe` (e.g. `npx serve publish-cpe`).

Quick login (prepAdmin / teacher tools)
---------------------------------------
Maintainers can still use the quick-login URL when needed (e.g. prepAdmin for hub editing):

  https://YOUR-CPE-SITE/index.html?prep_link=1&prep_pass=YOUR_ADMIN_PASSWORD&prep_name=YourName&prep_track=cpe

(Use the same pattern on EGE/FCE sites with prep_track=ege|fce.) Strip sensitive params from bookmarks after use.

The old **My courses** side rail has been removed from the UI for performance.

Name across domains
-------------------
Different domains = different browser storage. Students on the CPE domain only enter a name on that domain once. If you need the same display name on another domain, use a quick link with prep_name or the prep_carry_name flow (see js/prep-player-name-bridge.js).

Published tree
--------------
  The folder `course-cms/` is not copied into `publish-*` (it stays in the repo for optional local / maintainer use). Netlify builds therefore do not serve `/course-cms/`.

Local check
-----------
  node scripts/build-netlify-track.mjs ege
  npx serve publish-ege
