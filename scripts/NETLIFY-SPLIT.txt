Separate Netlify sites (same repo, lighter entry per course)
=============================================================

Why
---
- One huge index.html + auto-redirects between CPE / EGE / FCE can feel slow and
  "kick you" between login and hubs. Netlify is fine; splitting reduces noise.

This repo builds three folders:
  publish-cpe   — home "/" → index.html (English Mastery Pro only on this site)
  publish-ege   — home "/" → ege.html
  publish-fce   — home "/" → fce.html

Each folder gets a _redirects file and index.html meta prep-deploy-track so the
app does not auto-send CPE users to ЕГЭ/FCE on a CPE-only deploy.

Default site (main branch)
--------------------------
netlify.toml runs:  node scripts/build-netlify-track.mjs cpe
publish directory:  publish-cpe

Second site (ЕГЭ)
-----------------
Netlify → Build settings:
  Build command:   node scripts/build-netlify-track.mjs ege
  Publish directory: publish-ege

Third site (FCE)
----------------
  Build command:   node scripts/build-netlify-track.mjs fce
  Publish directory: publish-fce

Local check
-----------
  node scripts/build-netlify-track.mjs ege
  npx serve publish-ege

Passwords
---------
Unchanged in code. On a single-track site students only see one course on the
welcome screen if you deep-link or brand that site for one exam.

Cross-links between sites: each Netlify URL has its own localStorage, so a name
typed on the CPE site does not exist on the EGE site until it is copied over.

What the code does:
  • js/prep-player-name-bridge.js — reads ?prep_carry_name= on load and saves to
    english_mastery_perfect + prep-player-display-name-v1 (include on index, ege, fce).
  • My courses adds prep_carry_name when navigating (and prep_stay=1 for index).

If you use THREE separate Netlify sites (different *.netlify.app domains), set peer
origins once per browser context (Netlify “Snippet” or inline before rail):

  window.__PREP_PEER_ORIGINS__ = {
    cpe: "https://YOUR-CPE-SITE.netlify.app/",
    ege: "https://YOUR-EGE-SITE.netlify.app/",
    fce: "https://YOUR-FCE-SITE.netlify.app/"
  };

Without this object, course links stay same-origin (relative) and only work fully
on a single deploy that ships all HTML files together.

You can still use absolute URLs in ege.html / fce.html course-switch nav; add
&prep_carry_name=… only if you build those links in JS — the rail handles carry
automatically when __PREP_PEER_ORIGINS__ is set.
